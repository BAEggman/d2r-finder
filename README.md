# D2R 트레더리 아이템 파인더

디아블로 2 레저렉션 아이템을 **한글 이름**이나 **스크린샷**으로 검색해서, Traderie 매물 페이지로 바로 연결하고 현재 호가(시세)를 페이지 안에서 보여주는 정적 웹앱입니다. GitHub Pages에 그대로 올려서 쓸 수 있습니다.

## 구성 파일

| 파일 | 역할 |
|---|---|
| `index.html` | 앱 본체 (검색 UI + 스크린샷 인식 + 시세 표시, 외부 프레임워크 없음) |
| `data.js` | 아이템 사전 1,324개 — 유니크 407 · 세트 140 · 룬어 93 · 룬 33 · 베이스/기타 651. D2R 공식 한국어 로컬라이제이션 문자열 기반, 한↔영 매핑 + 커뮤니티 약칭(수수, 콜투, 조던, 샤코…) 포함 |
| `worker.js` | Cloudflare Worker 소스 — 시세 중계 + 스크린샷 인식(Gemini) 프록시 |
| `README.md` | 이 문서 |

기능별 필요 조건:

- **한글 이름 검색 → Traderie 링크**: `index.html` + `data.js`만 있으면 동작. 서버·키·설정 전부 불필요.
- **페이지 안 시세 표시**: 위 2개 + Cloudflare Worker 1개 필요 (아래 2번).
- **스크린샷으로 검색**: 위 Worker + 무료 Gemini API 키 1개 필요 (아래 3번).

## 1. GitHub Pages에 올리기

1. GitHub에서 새 저장소를 만듭니다 (예: `d2r-finder`, Public).
2. `index.html`, `data.js`, `README.md`를 저장소 루트에 업로드합니다. (`worker.js`도 같이 올려두면 관리가 편합니다 — Pages 동작에는 영향 없음)
3. 저장소 **Settings → Pages → Source**에서 `Deploy from a branch`, 브랜치 `main`, 폴더 `/ (root)` 선택 후 저장.
4. 1~2분 뒤 `https://<내계정>.github.io/d2r-finder/` 에서 접속됩니다.

여기까지만 해도 "한글 이름 → 트레더리 매물 페이지" 흐름은 완성입니다.

## 2. Cloudflare Worker 만들기 (5분) — 시세 연동용

브라우저 보안 정책(CORS) 때문에 GitHub Pages에서 traderie.com API를 직접 못 부릅니다. 무료 Worker가 중간에서 대신 전달해 줍니다.

1. https://dash.cloudflare.com 가입/로그인 (무료 플랜이면 충분, 카드 불필요).
2. 왼쪽 메뉴 **Workers & Pages → Create → Create Worker**.
3. 이름을 정하고(예: `d2r-proxy`) **Deploy**를 누른 뒤, **Edit code**로 들어갑니다.
4. 기본 코드를 전부 지우고 `worker.js` 내용을 통째로 붙여넣고 **Deploy**.
5. 화면에 표시되는 주소(예: `https://d2r-proxy.<내계정>.workers.dev`)를 복사합니다.
6. 앱 하단 **"시세 연동 설정"**을 펼쳐 그 주소를 붙여넣고 저장.

이제 아이템을 선택하고 **시세 불러오기**를 누르면 매물 호가·판매자·갱신 시각·옵션이 페이지 안에 표시됩니다.

권장: `worker.js` 상단의 `ALLOWED_ORIGIN`을 `"*"` 대신 본인 Pages 주소(`"https://<내계정>.github.io"`)로 바꿔서 남이 내 프록시를 못 쓰게 하세요. 무료 한도는 하루 10만 요청이라 개인 용도로는 넉넉하고, 같은 요청은 3분간 캐시되어 트레더리에 부담을 덜 줍니다.

## 3. 스크린샷 인식 켜기 — Gemini 무료 API

스크린샷 속 아이템은 Google Gemini가 알아봅니다. 키는 브라우저가 아니라 **Worker 시크릿**에 보관하므로 사이트 소스에는 노출되지 않습니다.

1. https://aistudio.google.com/apikey 에서 **Create API key** (무료, 카드 불필요).
2. Cloudflare 대시보드 → **Workers & Pages → d2r-proxy → Settings → Variables and Secrets**.
3. **Add** → 유형 **Secret** → 이름 `GEMINI_API_KEY` → 값에 발급받은 키를 붙여넣고 **Deploy**.

끝입니다. 앱의 "스크린샷으로 검색" 탭에서 이미지를 올리면 (드래그·`Ctrl+V` 붙여넣기도 됨) 아이템 영문명을 알아내 사전과 대조해 후보를 보여줍니다.

- 모델은 `gemini-2.5-flash-lite` → `gemini-2.5-flash` → `gemini-2.0-flash` 순으로 자동 시도합니다. 특정 모델을 쓰려면 같은 화면에서 일반 변수 `GEMINI_MODEL`을 추가하세요.
- 무료 한도는 모델별로 하루 수백~1,000회 수준이고 자정(태평양 시간)에 초기화됩니다. 개인 용도로는 넉넉합니다.
- 이미지는 브라우저에서 긴 변 1280px JPEG로 줄여 보내며, 저장하지 않습니다.
- `ALLOWED_ORIGIN`과 다른 주소에서 온 요청은 Worker가 거부하므로, 남이 이 인식 기능을 갖다 쓰긴 어렵습니다.

## 4. 문제 해결

- **"프록시 응답이 JSON이 아닙니다"** — Worker 주소 오타이거나, Worker에 코드가 제대로 배포되지 않은 경우가 대부분입니다. 브라우저에서 Worker 주소를 직접 열었을 때 `{"ok":true,...}`가 보여야 정상입니다.
- **봇 차단(Cloudflare) 안내가 뜰 때** — 트레더리가 자동 조회를 막은 상태입니다. 잠시 후 재시도하고, 계속되면 이 방식이 차단된 것일 수 있습니다. 이 경우에도 검색과 "Traderie에서 보기" 링크는 계속 동작합니다.
- **"아이템 ID를 찾지 못했습니다"** — 트레더리 쪽 표기가 살짝 다른 아이템입니다(변형판 등). 링크로 직접 확인하세요.
- **시세가 갑자기 안 나올 때** — 트레더리가 내부 API 구조를 바꿨을 가능성이 큽니다. traderie.com에서 F12(개발자 도구) → Network 탭을 열고 아이템 페이지를 새로고침하면 `/api/diablo2resurrected/...` 요청이 보입니다. 바뀐 경로·파라미터를 `index.html`의 `loadPrices` 부분에 반영하면 됩니다.
- **"Gemini API 키가 워커에 등록되어 있지 않습니다"** — 위 3번의 `GEMINI_API_KEY` 시크릿이 없거나 이름이 다릅니다. 대소문자까지 정확히 맞춰야 합니다.
- **"Gemini 무료 할당량을 다 썼습니다"** — 하루 한도를 넘겼습니다. 자정(태평양 시간) 이후 초기화됩니다.
- **인식 결과가 엉뚱할 때** — 아이템 설명창만 크게 잘라서 올리면 좋아집니다. 인식된 텍스트를 직접 고쳐 "다시 찾기"를 눌러도 되고, 이름 검색 탭이 가장 확실합니다.

## 5. 데이터 출처와 갱신

- 아이템 명칭: D2R 공식 로컬라이제이션 문자열(`item-names.json`, `item-runes.json`, `item-nameaffixes.json`)의 `koKR`/`enUS` 필드. 2.5 파괴 부적, 2.6 룬어(모자이크 등)는 국내 커뮤니티 표기로 교차 검증해 수동 보강.
- 유니크/세트 목록·베이스 매핑: 커뮤니티 게임 데이터(`uniqueitems.json`, `setitems.json`).
- 약칭 사전은 `data.js`를 열어 해당 아이템의 `"a":[...]` 배열에 단어를 추가하면 바로 검색에 반영됩니다.

## 6. 알아둘 점

- 시세 연동은 **트레더리의 비공식 API**를 사용합니다. 약관상 회색지대이며 언제든 막힐 수 있으니, 개인 조회 용도로만 가볍게 쓰는 것을 권장합니다. 대량·자동 반복 호출은 하지 마세요.
- 이 도구는 팬 유틸리티로, Blizzard 및 Traderie(Akrew)와 무관합니다. 아이템 명칭 등의 권리는 각 권리자에게 있습니다.
